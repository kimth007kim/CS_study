컬렉션의 구현 방법을 노출 시키지 않으면서도 그 집합체 안에 들어있는 모든 항목에 접근하는 방법

노출 모듈 패턴

자바스크립트에서 즉시 실행 함수를 통해 private,public 같은 접근 제어자를 만드는 것

단점 private 메소드 수정

각각의 역할을 나눠 코드를 관리하자, 역할을 나누어 관리하면 , 유지보수와 개발효율이 올라간다.

mvc 패턴
m(model): 어플리케이션에서 사용되는 데이터와 그데이터를 처리하는 부분(데이터베이스, 상수,변수)
v(view): 사용자에게 보여지느 ui 부분
c(controller): 사용자의 입력을 받고 처리하는 부분


단일책임의 원칙


장점: 가장 널리 사용되는 패턴 , 단순함,재사용성 확장성이 용이하다.
단점: view 와 model 사이의 의존성이높다.
예: 리액트

mvp 패턴
p(presenter) : view 가 요청한 정보로 model 을 가공하여 view 에전달해주는 부분 view와 model을 붙여주는 접착제 같은역할, 단위 테스트 가능
책임 분배의 구조 낮은 테스트 용이성

mvc패턴의 문제점이  컨트롤러에서 너무많은 작업이 있어서 사용한다.

장점: view 와 model 의 의존성 해결
단점: view 와 presenter 사이의 의존성 강화

mvvm:
vm: view Model you

비즈니스로직과 프레젠테이션 로직을 ui로 부터 분리하는것이 목적
테스트,유지보수 재사용이 쉬워진다.
