HTTP
	HyperText Transfer Protocol
	
	HTTP메세지에 모든 것을 전송한다.
	HTML,TEXT
	이메일,음성,영상,파일
	JSON,XML(API)
	거의 모든 형태의 데이터 전송 가능

	StateLess 장점
	무상태 프로토콜이다.(StateLess)
	장점:  불특정 다수를 대상으로 했을때 적합,접속유지를 최소한으로 할 수 있기때문에, 많은 유저의 요청 처리가능

	StateLess 단점
	연결을 끊어 버리기 때문에, 클라이언트의 이전상태를 알수 없다.
	로그인시에 로그인정보를 알수없다, 쿠키로 해결한다.


	비연결성 
	HTTP는 기본이 연결 유지하지않는 모델
	일반적으로 초 단위 이하의 빠른 속도로 응답
	서버 자원을 매우 효율적으로 사용할수 있다.(연결을 유지하는 비용이 들지않는다.)


	비연결성 단점
	TCP/IP연결을 새로 맺어져야함 - 3Way handshake를 매번해야함
	웹 브라우저로 사이트를 요청하면 HTML 뿐만아니라 자바스크립트, css, 추가 이미지 등 수많은 자원 함께 다운
	HTTP 지속 연결로 문제 해결(Persistent Connections)
